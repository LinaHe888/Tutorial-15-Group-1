# Submission Folder Guide

本目录用于按任务拆分组内材料，便于各成员分别上传与整理。

## 目录说明

- `task-1-decompilation/`：反编译、代码定位、关键截图、工具记录
- `task-2-system-model/`：系统理解、功能总结、系统模型图、关键资产与数据流
- `task-3-vulnerability-discovery/`：漏洞候选点、代码证据、随机数/密码学分析、主漏洞筛选
- `task-4-threat-model/`：攻击者模型、攻击路径、风险分析、威胁模型说明
- `task-5-report-and-presentation/`：正式报告、AI log、PoC、演示文稿、视频脚本
- `shared-assets/`：全组共享材料（模板、参考图、统一截图、公共说明）

## 使用建议

1. 每位组员优先将自己的材料上传到对应 task 文件夹。
2. 若文件会被多个任务共用，请放入 `shared-assets/`。
3. 提交前由一名整合负责人统一整理最终交付物。


## 命名与整理建议

为便于小组协作，建议采用统一命名规则：
- 截图：`taskX-topic-description.png`
- 草稿：`draft-v1`、`draft-v2`
- 最终文件：统一使用课程要求文件名
- 公共材料：优先归档至 `shared-assets/`
